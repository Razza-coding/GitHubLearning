These are common Git commands used in various situations:

start a working area (see also: git help tutorial)
[LEARNED]   clone             Clone a repository into a new directory
[LEARNED]   init              Create an empty Git repository or reinitialize an existing one

work on the current change (see also: git help everyday)
[LEARNED]   add               Add file contents to the index
[LEARNED]   mv                Move or rename a file, a directory, or a symlink
[LEARNED]   restore           Restore working tree files
[LEARNED]   rm                Remove files from the working tree and from the index
[LEARNED]   sparse-checkout   Initialize and modify the sparse-checkout

examine the history and state (see also: git help revisions)
[LEARNED]   bisect            Use binary search to find the commit that introduced a bug
[LEARNED]   diff              Show changes between commits, commit and working tree, etc
[LEARNED]   grep              Print lines matching a pattern
[LEARNED]   log               Show commit logs
[LEARNED]   show              Show various types of objects
[LEARNED]   status            Show the working tree status

grow, mark and tweak your common history
[LEARNED]   branch            List, create, or delete branches
[LEARNED]   commit            Record changes to the repository
[LEARNED]   merge             Join two or more development histories together
[       ]   rebase            Reapply commits on top of another base tip
[       ]   reset             Reset current HEAD to the specified state
[LEARNED]   switch            Switch branches
[LEARNED]   tag               Create, list, delete or verify a tag object signed with GPG

collaborate (see also: git help workflows)
[LEARNED]   fetch             Download objects and refs from another repository
[LEARNED]   pull              Fetch from and integrate with another repository or a local branch
[LEARNED]   push              Update remote refs along with associated objects
